# Synthesis Interpretation — synth_top (compute_core)
## Codefest 7 — ECE 510 Spring 2026
## Tool: OpenLane 1.1.1 | PDK: Sky130A (bdc9412b) | Clock: 10.0 ns

---

## Clock Period and Worst-Case Slack

The synthesis was run with a clock period of 10.0 ns (100 MHz) on the
Sky130A process node. The synthesis exploration produced two strategy
families — AREA and DELAY. The best area result (AREA_0) achieved a
critical path delay of 2190.0 ps, giving a worst-case slack of
10000 - 2190 = **+7810 ps**. The best delay result (DELAY_0) achieved
3090.0 ps, giving a slack of **+6910 ps**. Both strategies meet timing
comfortably at 100 MHz with significant positive slack, confirming the
design is timing-clean at this clock frequency.

---

## Critical Path

The critical path in the AREA_0 strategy runs through the combinational
multiply-accumulate logic in the `av_unit` stage, which performs a
32-bit × 32-bit multiplication followed by an arithmetic right shift.
The dominant cell types along the critical path are NAND2 (195 instances),
NOR2 (169 instances), AND2 (101 instances), and XOR2/XNOR2 (200 combined
instances). These are characteristic of a carry-chain adder and Wallace
tree multiplier structure generated by Yosys for the MAC computation.
The source register is the `relu_out` flip-flop in `relu_approx` and the
sink register is the `av_accum` flip-flop in `av_unit`.

---

## Total Cell Area and Top Contributors

The AREA_0 synthesis strategy produced a chip area of **23,189.74 µm²**
with **2,418 total cells**. The top three contributors by instance count are:

- `sky130_fd_sc_hd__nand2_2`: 195 instances — primary logic for adder carry chain
- `sky130_fd_sc_hd__nor2_2`: 169 instances — complement logic for MAC tree
- `sky130_fd_sc_hd__buf_1`: 156 instances — signal buffering across pipeline stages

The design contains exactly **149 `dfxtp_2` flip-flops**, corresponding
to: 32-bit `qk_accum` (32 FFs) + 32-bit `scaled` (32 FFs) + 32-bit
`relu_out` (32 FFs) + 32-bit `av_accum` (32 FFs) + 4 valid/control bits
(4 FFs) = 132 FFs minimum, with 17 additional pipeline registers.

## Warnings and Constraints

The CHECK pass reported **0 problems** for both AREA_0 and DELAY_0
strategies. Two warnings were noted: PNR_SDC_FILE and SIGNOFF_SDC_FILE
are not set, meaning no custom SDC timing constraints were provided —
the tool used its default base SDC. No hold violations, no unresolved
references, and no unsynthesizable constructs were found. The design
is fully synthesizable as submitted.

## How to Reproduce Synthesis

**Tool:** OpenLane 1.1.1 (efabless/openlane Docker image)
**PDK:** Sky130A (bdc9412b3e468c102d01b7cf6337be06ec6e9c9a)

Note: OpenLane 2 pip installation failed on Windows due to missing
Microsoft C++ Build Tools for the klayout dependency. OpenLane 1.1.1
was used instead — it uses the same Yosys + OpenSTA backend.
Professor approved this approach.

**Step 1 — Install Docker Desktop**
Download from https://www.docker.com/products/docker-desktop

**Step 2 — Pull OpenLane image:**
```
docker pull efabless/openlane:latest
```

**Step 3 — Download Sky130A PDK:**
```
docker run --rm -v "openlane_pdk:/root/.volare" efabless/openlane:latest sh -c "python3 -m volare fetch --pdk sky130 bdc9412b3e468c102d01b7cf6337be06ec6e9c9a"
```

**Step 4 — Run synthesis exploration:**
```
docker run --rm \
  -v "<path_to_repo>/codefest/cf07:/openlane1/designs/synth_top" \
  -v "openlane_pdk:/root/.volare" \
  efabless/openlane:latest \
  sh -c "cd /openlane1 && ./flow.tcl -design designs/synth_top -tag synth_run -synth_explore"
```

**Step 5 — Reports location:**
```
codefest/cf07/runs/synth_run/reports/synthesis/
```
